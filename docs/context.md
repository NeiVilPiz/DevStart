# Context API

## ¿Qué es Context?

Context API permite compartir estado entre componentes sin necesidad de pasar props manualmente.

---

## Uso potencial en el proyecto

En DevLaunch se podría utilizar Context para:

- almacenar historial de ideas generadas
- gestionar estado global de proyectos
- controlar configuraciones de usuario

---

## Provider

Un Provider permitiría envolver la aplicación y compartir datos globales.

---

## Cuándo es útil

Context es útil cuando:

- varios componentes necesitan los mismos datos
- el estado es global en la aplicación
- se quiere evitar prop drilling

---

## Implementación futura

En futuras versiones del proyecto se podría implementar Context para mejorar la escalabilidad de la aplicación.
